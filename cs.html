Acoustic Ray Modeling Simulation (ARMS)

A physically modeled acoustic room simulation that demonstrates how rooms attenutate sound based on different materials, microphone and speaker settings.

Key features:
   - .wav file I/O for digitally processing samples to simulate them within given rooms
   - Physical ray modeled reverberance
   - Schroeder (Sabine Formula) modeled reverberance
   - Built using SFML to visualize rooms and their physically modeled rays

   - Custom scripting system with features such as:
       - Custom material creation
       - Custom room generation
       - Customizable microphone settings
       - Customizable speaker settings

WebSynth

A browser-based polyphonic synthesizer using p5.js that features four distinct waveforms and real-time visualization. While initially developed as a tool to help teach high school students the basics of programming, it evolved into a personal project exploring audio synthesis through visualization.

Key features:
   - 4 selectable waveforms
   - Polyphonic playback for chords and harmonies
   - Interactive drag and drop GUI
   - MIDI-style input using a standard computer keyboard
   - Live visualization of frequency, amplitude, and MIDI note data

A key technical challenge I faced during this project was attempting to mitigate the audible 'clicks' from oscillators as they started and stopped While I implemented solutions such as amplitude ramping and ADSR envelopes, I found that the key issue lay within the lack of granular control given through the p5 framework. Through this experience I have learned valuable information in synthesis visualization and look forward to re-implementing this through C++ and portaudio and/or juce in the future to gain greater control over the audio stream and produce seamless output.
